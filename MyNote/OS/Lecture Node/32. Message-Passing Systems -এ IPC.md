Section 3.5-এ আমরা দেখিয়েছি কীভাবে cooperating processes একটি shared-memory environment-এ communicate করতে পারে। এই scheme-এ প্রয়োজন হয় যে processes-গুলো একটি memory region share করবে এবং shared memory access ও manipulate করার code application programmer-কে explicitly লিখতে হবে। একই প্রভাব অর্জনের আরেকটি উপায় হলো operating system নিজেই এমন একটি mechanism প্রদান করা, যার মাধ্যমে cooperating processes একটি message-passing facility ব্যবহার করে একে অপরের সঙ্গে communicate করতে পারে।

```c
item next_consumed;
while (true) {
    while (in == out); /* do nothing */
    
    next_consumed = buffer[out];
    out = (out + 1) % BUFFER_SIZE;
    /* consume the item in next_consumed */
}
```

**Chapter 3 Processes**

Message passing এমন একটি mechanism প্রদান করে যা processes-গুলোকে একই address space share না করেই communicate করতে এবং তাদের actions synchronize করতে দেয়। এটি বিশেষভাবে একটি distributed environment-এ উপযোগী, যেখানে communicating processes বিভিন্ন computer-এ থাকতে পারে এবং network-এর মাধ্যমে সংযুক্ত থাকে। উদাহরণস্বরূপ, একটি Internet chat program এমনভাবে design করা যেতে পারে যাতে chat participants একে অপরের সঙ্গে messages exchange করার মাধ্যমে communicate করে।

একটি message-passing facility কমপক্ষে দুটি operation প্রদান করে:
```c
send(message);
///and
receive(message);
```

একটি process দ্বারা পাঠানো messages fixed-sized অথবা variable-sized হতে পারে। যদি কেবল fixed-sized messages পাঠানো যায়, তবে system-level implementation তুলনামূলকভাবে straightforward হয়। তবে এই restriction প্রোগ্রাম লেখার কাজকে আরও difficult করে তোলে। বিপরীতভাবে, variable-sized messages-এর ক্ষেত্রে system-level implementation আরও complex হয়, কিন্তু programming task সহজ হয়ে যায়। এটি operating-system design জুড়ে দেখা যাওয়া একটি সাধারণ ধরনের trade off।

যদি processes **P** এবং **Q** একে অপরের সঙ্গে communicate করতে চায়, তবে তাদের পরস্পরের কাছে messages send এবং receive করতে হবে; অর্থাৎ তাদের মধ্যে একটি communication link বিদ্যমান থাকতে হবে। এই link বিভিন্ন উপায়ে implement করা যেতে পারে। এখানে আমরা link-এর physical implementation (যেমন shared memory, hardware bus, বা network—যেগুলো Chapter 19-এ আলোচনা করা হয়েছে) নিয়ে নয়, বরং এর logical implementation নিয়ে আলোচনা করছি। Link এবং `send()` / `receive()` operations logically implement করার কয়েকটি পদ্ধতি নিচে দেওয়া হলো:

- Direct অথবা indirect communication
    
- Synchronous অথবা asynchronous communication
    
- Automatic অথবা explicit buffering
