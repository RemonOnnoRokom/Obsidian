**Indirect communication**-এর ক্ষেত্রে, messages গুলো **mailboxes** অথবা **ports**-এর মাধ্যমে পাঠানো ও গ্রহণ করা হয়। একটি **mailbox**-কে abstractভাবে এমন একটি **object** হিসেবে দেখা যায়, যেখানে processes messages রাখতে পারে এবং যেখান থেকে messages সরানো যায়। প্রতিটি mailbox-এর একটি **unique identification** থাকে। উদাহরণস্বরূপ, **POSIX message queues** একটি mailbox চিহ্নিত করতে একটি **integer value** ব্যবহার করে। একটি process একাধিক ভিন্ন mailbox-এর মাধ্যমে অন্য একটি process-এর সঙ্গে communicate করতে পারে, কিন্তু দুইটি process কেবল তখনই communicate করতে পারে, যখন তাদের একটি **shared mailbox** থাকে।

এই ক্ষেত্রে `send()` এবং `receive()` primitives-গুলো নিম্নরূপভাবে সংজ্ঞায়িত করা হয়:

- `send(A, message)` — mailbox **A**-তে একটি message পাঠানো।
    
- `receive(A, message)` — mailbox **A** থেকে একটি message গ্রহণ করা।
    

এই scheme-এ একটি **communication link**-এর নিম্নলিখিত বৈশিষ্ট্য রয়েছে:

- একটি pair of processes-এর মধ্যে link কেবল তখনই establish হয়, যখন উভয় process-এর একটি **shared mailbox** থাকে।
    
- একটি link দুইটির বেশি processes-এর সঙ্গে associated থাকতে পারে।
    
- প্রতিটি pair of communicating processes-এর মধ্যে একাধিক ভিন্ন link থাকতে পারে, যেখানে প্রতিটি link একটি mailbox-এর সঙ্গে corresponding।
    

এখন ধরুন, processes **P1**, **P2**, এবং **P3**—তিনটিই mailbox **A** share করে। Process **P1**, mailbox **A**-তে একটি message পাঠায়, আর একই সময়ে **P2** এবং **P3** উভয়েই mailbox **A** থেকে `receive()` execute করে। প্রশ্ন হলো—**P1** পাঠানো message-টি কোন process receive করবে? এর উত্তর নির্ভর করে আমরা নিচের কোন method বেছে নিচ্ছি তার উপর:

- একটি link সর্বোচ্চ দুইটি process-এর সঙ্গে associated থাকতে দেওয়া।
    
- একই সময়ে সর্বোচ্চ একটি process-কে `receive()` operation execute করতে দেওয়া।
    
- system-কে arbitraryভাবে নির্ধারণ করতে দেওয়া কোন process message receive করবে (অর্থাৎ **P2** অথবা **P3**—কিন্তু দুজনই নয়—message receive করবে)। system কোন process message receive করবে তা নির্ধারণ করার জন্য একটি algorithm define করতে পারে (যেমন **round robin**, যেখানে processes পালাক্রমে message receive করে)। system প্রয়োজনে sender-কে receiver-এর পরিচয় জানাতে পারে।
    

একটি **mailbox** হয় কোনো **process**-এর মালিকানাধীন হতে পারে, অথবা **operating system**-এর মালিকানাধীন হতে পারে।  
যদি mailbox কোনো process-এর মালিকানাধীন হয় (অর্থাৎ mailbox-টি process-এর **address space**-এর অংশ হয়), তাহলে আমরা **owner** (যে কেবল এই mailbox-এর মাধ্যমে messages receive করতে পারে) এবং **user** (যে কেবল mailbox-এ messages send করতে পারে) —এই দুইটির মধ্যে পার্থক্য করি। যেহেতু প্রতিটি mailbox-এর একটি unique owner থাকে, তাই এই mailbox-এ পাঠানো message কোন process receive করবে তা নিয়ে কোনো confusion থাকে না। যখন যে process mailbox-এর owner, সেই process terminate করে, তখন mailbox-টিও অদৃশ্য হয়ে যায়। এরপর যদি কোনো process এই mailbox-এ message পাঠানোর চেষ্টা করে, তাহলে তাকে জানিয়ে দেওয়া হয় যে mailbox-টি আর exist করে না।

অন্যদিকে, যে mailbox **operating system**-এর মালিকানাধীন, তার নিজস্ব একটি existence থাকে। এটি independent এবং কোনো particular process-এর সঙ্গে attached থাকে না। তখন operating system-কে এমন একটি mechanism প্রদান করতে হয়, যা একটি process-কে নিম্নলিখিত কাজগুলো করতে দেয়:

- একটি নতুন mailbox create করা।
    
- mailbox-এর মাধ্যমে messages send এবং receive করা।
    
- একটি mailbox delete করা।
    

যে process একটি নতুন mailbox create করে, সে defaultভাবে সেই mailbox-এর **owner** হয়। শুরুতে, owner-ই একমাত্র process যে এই mailbox-এর মাধ্যমে messages receive করতে পারে। তবে, appropriate **system calls**-এর মাধ্যমে ownership এবং receiving privilege অন্যান্য processes-এর কাছে transfer করা যেতে পারে। অবশ্যই, এর ফলে প্রতিটি mailbox-এর জন্য একাধিক receiver থাকতে পারে।
